# Architecture Decision Records (ADRs)

This directory contains Architecture Decision Records for the pathfinder Chrome Extension.
Each ADR documents a significant technical decision with context, options considered, and rationale.

## Index

| ADR | Title | Status |
|-----|-------|--------|
| [ADR-001](adr-001-manifest-v3.md) | Chrome Manifest V3 with Vite + CRXJS | Accepted |
| [ADR-002](adr-002-client-side-only.md) | Fully Client-Side Architecture (BYOK) | Accepted |
| [ADR-003](adr-003-indexeddb-vectors.md) | IndexedDB for Vector Storage | Accepted |
| [ADR-004](adr-004-ai-provider-abstraction.md) | Multi-Provider AI Abstraction Layer | Accepted |
| [ADR-005](adr-005-self-healing.md) | Self-Healing Test Selectors Strategy | Accepted |

## Format

Each ADR follows this structure:
- **Status**: Proposed / Accepted / Deprecated / Superseded
- **Context**: Why this decision was needed
- **Decision**: What was decided
- **Options Considered**: Alternatives evaluated
- **Consequences**: Trade-offs accepted
