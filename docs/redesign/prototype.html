coverage model — five axes
1. Structural
Every interactive element visited at least once. BFS over CDP accessibility tree + DOM, priority by a11y role and viewport visibility. Shadow DOM pierced. iframes descended. Hover-revealed elements captured via synthetic mouseMoved sweeps.
2. State
DOM-signature hashing after each interaction; when hash is new, the state is added to the map. Keeps the graph small and guarantees we exercise distinct states (open/closed modals, toggled switches, sorted tables).
3. Data
Form fuzzing driven by constraint inference: HTML5 attrs + pattern/min/max + placeholder hints + Zod-style schemas inferred from API contracts (if present). Always generates boundary, empty, unicode, and injection-ish inputs.
4. Flow
Sequences from graph traversal plus LLM-proposed scenarios ("what would a user do next?"). Sequences are replayable and de-duped by canonical path.
5. Regression
Every run diffs against last-green baseline per node (HAR, a11y tree, visual, console, timings). Anomalies become findings automatically.
how it actually handles tricky UI
modalsdetect via aria-modal, role=dialog, z-index + body scroll-lock. Test Escape, outside-click, close button. Focus-trap probe.
iframesenumerate via Target.attachToTarget; run agent recursively in each same-origin frame. Cross-origin: assert boundary + load success + CSP only.
shadow DOMDOM.getDocument(pierce:true). Treat shadow roots as normal subtrees. Works for design-system web components.
toggles / switchesdistinguish from checkboxes via role=switch / aria-checked. Capture network side-effects on toggle. Test both directions + persistence across reload.
hoversInput.dispatchMouseEvent mouseMoved with dwell. Watch for DOM mutations + tooltip-like elements appearing. Measure delay + dismissal.
drag / resizesequenced mousedown → move steps → mouseup. Detect handles via cursor style + draggable.
virtualised listsscroll with waits; fingerprint row keys not indices. Avoid infinite scroll traps with a visited-set.
uploads / filesPage.handleFileChooser with synthetic files from a sandboxed fixture set.
auth / SSOsession snapshot (cookies + storage) captured once per run. Never replay credentials through the UI after first login.
spa routinglisten on Page.frameNavigated + History API hooks; treat URL+DOM-hash as the node key.
engine architecture (fresh build — no jirashastra, no enggxmcp)
1. Event log (source of truth)
Append-only Observation events in IndexedDB + optional SQLite mirror in a local sidecar. Everything is a projection: map, graph, flows, HAR view, a11y view.
2. Capability workers
Crawler, explorer, executor, analyser are peers on the log. Run concurrently. Each is pausable/resumable. No "pipeline stage" coupling.
3. Deterministic shell
AI proposes. Rules layer validates, dedupes, and gates irreversible actions (delete, pay, send). Nothing destructive ships without a check evaluated deterministically.
4. Local-first sidecar
Tiny native host (Go or Node) for SQLite, vector store, and long-running agent loops. Talks to extension over native messaging. No external MCP dependency — purpose-built protocol, just JSON-RPC.
5. CDP-only automation
All interaction is CDP (not content-script clicks). Gives us shadow DOM piercing, iframe descent, input simulation fidelity, HAR, a11y tree, trace events, screenshots — in one surface.
6. Model router
Haiku for map/hover probes (cheap, fast). Sonnet for flow & check generation. Opus for root-cause analysis. Budget-aware; degrades under cost cap.
